# Project Governance

## Overview

KushCloud is an open-source project maintained by [Linacre](https://github.com/LIN4CRE). This document outlines the governance model for the project, including decision-making processes, roles, and responsibilities.

## Roles

### Maintainers
Maintainers are individuals who have demonstrated a long-term commitment to the project and have been granted write access to the repository. They are responsible for:
- Reviewing and merging Pull Requests.
- Managing project releases.
- Setting the technical direction and roadmap.
- Upholding the Code of Conduct.

Current Maintainers:
- [Linacre](https://github.com/LIN4CRE)

### Contributors
Contributors are individuals who contribute code, documentation, or other improvements to the project. Anyone can become a contributor by submitting a Pull Request.

## Decision Making Process

### Technical Decisions
Minor technical decisions are made through the regular Pull Request review process. Larger decisions or significant changes to the architecture should be discussed in a GitHub Issue or Discussion first to reach a consensus.

### Conflict Resolution
In the event of a disagreement, the maintainers will work together to find a solution that best serves the project's goals. If a consensus cannot be reached, the Project Lead ([Linacre](https://github.com/LIN4CRE)) has the final say.

## Becoming a Maintainer

New maintainers are appointed by existing maintainers based on their consistent contributions, community involvement, and alignment with the project's values.

## Open Source Philosophy

KushCloud is committed to being a transparent and inclusive project. We value community feedback and encourage everyone to participate in its development.
